# COMPREHENSIVE VAULT PATHS ANALYSIS
# For kss and kcs Kubernetes clusters

## SUMMARY OF FINDINGS

1. SHARED PATHS (SAME FOR BOTH CLUSTERS)
   - cloudflare (Cloudflare API token)
   - keycloak/admin (Keycloak admin password)
   - keycloak/broker-client (Upstream IdP federation client secret)
   - keycloak/argocd-client (ArgoCD OIDC client secret)
   - keycloak/oauth2-proxy-client (OAuth2-Proxy OIDC client credentials)
   - keycloak/grafana-client (Grafana OIDC client secret)
   - keycloak/jit-service (JIT service client secret)
   - keycloak/test-users (Test user passwords)
   - keycloak/db-credentials (Keycloak PostgreSQL credentials)
   - oauth2-proxy (OAuth2-Proxy cookie secret)
   - grafana/admin (Grafana admin credentials)
   - harbor/admin (Harbor admin credentials)

2. PER-CLUSTER PATHS (DIFFERENT FOR EACH CLUSTER)
   - minio/loki-kss (Loki MinIO credentials for kss cluster)
   - minio/loki-kcs (Loki MinIO credentials for kcs cluster)

## CONFLICT ANALYSIS

POTENTIAL CONFLICTS:
- minio/loki-* paths are correctly per-cluster
- All other Keycloak-related paths appear to be shared

RISKS:
- If kcs cluster tries to deploy Keycloak with different db credentials, 
  it will overwrite kss cluster's keycloak/db-credentials
- If different clusters need different Grafana credentials, 
  grafana/admin will conflict
- If different clusters need different ArgoCD configurations,
  keycloak/argocd-client will conflict
